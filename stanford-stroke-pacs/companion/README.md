# SSC Companion App

React frontend + FastAPI backend for multi-level annotation workflows with an
embedded OHIF preview pane.

## Architecture

- **Backend**: FastAPI (Python) serves the REST API on port **8043** and the
  built React frontend from `dist/`.
- **Frontend**: React + Vite + Tailwind CSS. Vite compiles everything into
  static files in `dist/`; no Node.js process runs in production.
- **Viewer integration**: the companion resolves OHIF viewer URLs through the
  backend and can embed OHIF in a lower preview pane inside the main app.

## Current UI

- **Top bar**: home link, level switcher (`Patients / Studies / Series`),
  column selector, Reset View button, label-definition button, and auth
  controls.
- **Sidebar**: search, label filters grouped by level, and modality quick
  filters for study/series browsing.
- **Main table**: one generic data table supports patients, studies, and
  series, including nested expansion (`patient -> study -> series`) and inline
  annotation editing. Column headers are sticky during vertical scroll.
  The first column can be frozen (pin button in the header) for horizontal
  scroll. Columns can be reordered by dragging and dropping headers.
  All table preferences (column visibility, order, sort, filters, frozen
  state) are persisted per user on the server and restored on next visit.
- **OHIF preview pane**: when active, the lower portion of the page embeds OHIF
  in an `iframe`. Study clicks preview the whole study; series clicks open a
  series-specific OHIF URL for that study.
- **OHIF action buttons**: study and series rows expose explicit `OHIF`
  buttons for opening the viewer in a separate tab (hidden at patient level).

Users always access the app at `http://<host>:8043`.

## Prerequisites

- Python 3.11+ with the `pacs` conda environment (or equivalent virtualenv)
- Node.js 20+ and npm (for building the frontend)
- PostgreSQL with the `stanford-stroke` database
- `.env` file in the parent directory (`stanford-stroke-pacs/.env`)

## Production

Build the frontend and start the server:

```bash
cd companion

# Install frontend deps (first time or after package.json changes)
npm install

# Build React app into dist/
npm run build

# Start FastAPI (uses the pacs conda environment)
conda activate pacs
uvicorn app:app --host 0.0.0.0 --port 8043
```

Or use the build helper:

```bash
./build.sh                # runs npm run build
```

### systemd (auto-start on boot)

A systemd unit file is provided at the repo root:

```bash
sudo cp ssc-companion.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now ssc-companion
```

After rebuilding the frontend:

```bash
npm run build
sudo systemctl restart ssc-companion
```

Check status:

```bash
sudo systemctl status ssc-companion
sudo journalctl -u ssc-companion -f
```

## Development (with hot-reload)

For frontend work, run the FastAPI backend and the Vite dev server separately.
The Vite dev server proxies `/api` requests to the backend automatically.

```bash
# Terminal 1 — FastAPI backend with auto-reload
conda activate pacs
cd companion
uvicorn app:app --port 8043 --reload

# Terminal 2 — Vite dev server (hot-reload for React/CSS changes)
cd companion
npm run dev
```

Then open **http://localhost:5173** in the browser. Vite serves the frontend
with instant hot-reload; API calls are proxied to `:8043` transparently.

When done, build for production and restart:

```bash
npm run build
sudo systemctl restart ssc-companion
```

> **Note**: The Vite dev server on port 5173 is only for development
> convenience. In production, only port 8043 is used — FastAPI serves both the
> API and the built frontend.

## Project Layout

```
companion/
  app.py              FastAPI backend (API + serves dist/)
  requirements.txt    Python dependencies
  package.json        Frontend dependencies (React, Vite, Tailwind)
  vite.config.js      Vite config with /api proxy for dev
  index.html          Vite HTML entry point
  build.sh            Helper to rebuild frontend
  src/
    main.jsx          React entry point
    app.css           Tailwind CSS entry (@theme with brand colors)
    App.jsx           React Router (/ and /app)
    api/client.js     Fetch wrapper with 401 handling
    context/          AuthContext (login/logout/session state)
    pages/            Landing and Companion page components
    components/       TopBar, Sidebar, DataTable, PreviewPane, InlineEdit, etc.
  dist/               Built frontend (generated by npm run build)
```
