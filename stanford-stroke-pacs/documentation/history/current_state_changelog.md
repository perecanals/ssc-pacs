# Historical changelog (from former `current_state.md`)

**Purpose:** Dated record of major documentation/system changes previously maintained in a single “current state” file. **Not** a substitute for git history or live code. For current architecture see [`../reference/architecture.md`](../reference/architecture.md).

When you make significant product or doc changes, prefer updating the relevant modular doc plus git commits; append here only if you want a running human-readable log.

---

| Date | Summary |
|------|---------|
| 2026-03-04 | Initial current_state.md created reflecting full system: Orthanc + OE2 + OHIF + Companion with JWT auth, inline editing, label definitions, column visibility, and landing page. |
| 2026-03-05 | DB-backed user management: users stored in `users` (bcrypt). `manage_users.py` replaces manual credential editing. Orthanc secrets removed from `orthanc.json` and moved to `.env` plus `orthanc_users.json`. `init_orthanc_db.sh` replaced the earlier SQL bootstrap. Companion login validates against DB instead of Orthanc API. |
| 2026-03-17 | Stanford Stroke Center adaptation: renamed all FlowCat references to SSC. Database changed from `proces` to `stanford-stroke`. Source tables changed from `flowcat_image_series` to `image_series` + `image_study`. Columns renamed: `nhc` -> `patient_id`, `seriesdescription_` -> `seriesdescription`, `dicomdir_path` -> `dicom_dir_path`. Dropped `proces_id`. Companion tables renamed: `flowcat_annotations` -> `annotations`, `flowcat_label_definitions` -> `label_definitions`, `flowcat_users` -> `users`. Containers renamed: `flowcat-orthanc` -> `ssc-orthanc`, `flowcat-companion` -> `ssc-companion`. |
| 2026-03-18 | Companion redesign: replaced monolithic static HTML with React + Vite + Tailwind CSS frontend (component-based, in `companion/src/`). Removed Docker packaging for companion; now runs natively via systemd (`ssc-companion.service`). FastAPI backend preserved with identical API contracts; now loads `.env` via python-dotenv and serves Vite build output from `dist/`. Removed companion from `docker-compose.yml`. Added `companion/build.sh`, `companion/README.md`. |
| 2026-03-18 | Multi-level annotation system: extended companion to support patient, study, and series level annotations. Added `level` column to `annotations` and `label_definitions` tables with partial unique indexes per level. Relaxed `NOT NULL` on `seriesinstanceuid`/`studyinstanceuid` in `annotations`. Added new backend endpoints: `GET /api/patients`, `GET /api/studies`, `GET /api/patients/{id}/studies`, `GET /api/studies/{uid}/series`. Added cross-level label filtering via `_label_filter_sql` helper and `label_level` query parameter. Added inherited annotation attachment for parent-level labels. Added snapshot refresh endpoint (`POST /api/snapshots/refresh`) for `snapshot_patients`, `snapshot_studies`, `snapshot_seriess`. Added `select` datatype with `options` column to `label_definitions`. Frontend: replaced `SeriesTable` with generic `DataTable` supporting all three levels, level tabs in `Companion.jsx`, expandable nested rows (patient→study→series), cross-level column selector, all-level sidebar label display, inline editing with `stopPropagation`, live subtable refresh after mutations, scroll-limited series subtables. Component styles extracted to co-located `.css` files. |
| 2026-03-18 | Companion OHIF preview integration: moved the level switcher and table controls into the top bar, added an embedded lower OHIF preview pane, and added active-row highlighting for study/series selections. Study rows can expand and preview together; series selections can request series-filtered OHIF URLs. The preview pane can be collapsed and reopened via row selection, and study/series action buttons still open OHIF in a separate tab. |
| 2026-03-24 | Storage-root migration: moved the canonical imaging tree from `/home/perecanals/pacs/imaging_database` to `/DATA2/pacs_imaging_data`. Updated the Orthanc bind mount, legacy image integration protocol base directory, and deployment documentation to use the new path. |
| 2026-03-24 | Companion column-visibility refinement: when a sidebar label filter is activated, the corresponding label column is forced visible in `DataTable` so the filtered attribute is always shown. Clearing the label filter now removes that temporary column unless it was already part of the user's saved visible-column preferences. |
| 2026-03-24 | Companion cleanup: removed the unused legacy `SeriesTable` component and its stylesheet after confirming the live `/app` route renders the generic `DataTable` instead. |
| 2026-03-24 | Labelled-table refresh bugfix: fixed the full rebuild helper so `POST /api/labelled-tables/refresh` no longer crashes with a `TypeError` when computing update columns. |
| 2026-03-31 | Companion table UX improvements: added sticky column headers (pinned `thead` during vertical scroll), a toggleable frozen first column (pin button in the first header cell), and drag-and-drop column reordering via native HTML5 DnD. Fixed container overflow so the table scrolls within its container instead of overflowing past the sidebar. Switched table from `border-collapse` to `border-separate border-spacing-0` to enable sticky positioning. Removed the empty Actions column from the patient-level table. |
| 2026-03-31 | Server-side user preferences: replaced all `localStorage`-based table state persistence with a `user_preferences` SQL table. Column visibility, column order, sort, filters, and frozen-column state are now persisted per user and per level via `GET/PUT /api/preferences/{level}`. Preferences are loaded on mount and saved with a debounced PUT (immediate flush on unmount/tab close). Added a "Reset View" button in the top bar to restore default table state. |
| 2026-03-31 | Shared annotations: changed annotation uniqueness from per-user `(entity_id, label, created_by)` to shared `(entity_id, label)`. Migration deduplicates existing per-user annotations (keeping the most recent). Upserts now overwrite the single shared value and update `created_by` to the last editor. Frontend `InlineEdit` shows one shared value for all users instead of splitting into "my annotation" vs "others"; `created_by` is shown as a tooltip. |
| 2026-04-09 | **Cold storage / hot cache (optional):** Added `dicom_archive_path` on `image_series`, tables `cache_state` and `orthanc_resource_map`, `companion/cache_manager.py` (warm study → hot cache, Orthanc `POST /instances`, TTL eviction). New API: `GET /api/storage-mode`, `POST /api/studies/{uid}/warm` (auth), `GET .../cache-status`. `mode = "cold_cache"` in `config.toml` branches OHIF resolver and series download (stream `.tar.zst` when available). Frontend warms via `warmOhif.js` before OHIF. Evaluation script `benchmarks/cold_storage_evaluation.py`; offline archiver `scripts/archive_all_series.py` targeting `/DATA2/pacs_imaging_data_compressed`. `.env.example` (secrets), `config.toml` (paths/mode), and `docker-compose.yml` document switching Orthanc `/dicom-data` mount between legacy tree and hot cache. See `documentation/cold_storage/design.md` and `documentation/cold_storage/runbook.md`. |
| 2026-04-09 | Documentation modularization: split monolithic `current_state.md` into `reference/`, `guides/`, `operations/`, `cold_storage/`, and `history/`; added `context.md`. |
| 2026-05-13 | **User-system consolidation:** the PostgreSQL `users` table is now the single source of truth for end-user authentication. Companion reverse-proxies `/ohif/*` and `/dicom-web/*` to Orthanc (`companion/routes/proxy.py`, async `httpx.AsyncClient` with streaming), attaching the service-account Basic auth from `.env` on every upstream call. End users no longer have entries in `orthanc_users.json` — only the service account and admin users (`users.is_admin=True`) remain there so admins can reach `:8042/ui/app/` and `:8042/ohif/` directly. `scripts/admin/manage_users.py` regular `add`/`passwd`/`remove` only touch the DB; they also mirror admin users into `orthanc_users.json`. A new `rotate-service-account` subcommand atomically rewrites `ORTHANC_ADMIN_PASSWORD` in `.env` and the matching JSON entry. The `/api/me` endpoint now returns `{"username", "is_admin"}` and `require_admin` was promoted into `companion/auth.py` as a shared dependency. Frontend Landing repoints OHIF to relative `/ohif/`, hides the "Orthanc Explorer" card for non-admins, and points it at direct `:8042/ui/app/` for admins. Old tabs cached against `:8042/ohif/...` should be closed/force-refreshed. |
| 2026-05-13 | **Login page + idle-expiry flow:** added a dedicated `/login` route and `ProtectedRoute` wrapper so `/` (Landing) and `/app` (Companion) require authentication; unauthenticated visits redirect to `/login`. Companion's `TopBar` lost its inline login form (the route is now strictly post-login). Landing gained a thin top-bar identity strip (`Logged in as <user> · Log out`); the footer port hint was removed. `AuthContext` now owns a `wasAuthedRef` so the "Your session expired. Please sign in again." banner on `/login?expired=1` appears only on involuntary expiry — fresh visits and intentional logouts route to `/login` with no banner. Sliding idle TTL shortened from 15 min to 5 min (`config.toml`'s `session_timeout_hours = 0.0833`); the absolute 24 h cap is unchanged. |
